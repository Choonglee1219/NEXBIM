# NEXBIM 사용자 설명서

## 1. 개요
* 본 애플리케이션은 **OpenBIM Components**와 **Oracle Database**를 기반으로 구축된 웹 기반 BIM(Building Information Modeling) 뷰어 및 협업 플랫폼임.
* IFC/Fragment 모델의 시각화, 데이터베이스 연동 저장, BCF이슈 관리, 실시간 간섭 검토(Clash Detection), 공간구조 및 속성 데이터 조회 기능을 통합 제공함.

---

## 2. Projects Manager
* 애플리케이션의 시작 화면으로, 사용자의 프로필 관리, 등록된 프로젝트 목록의 필터 조회, 신규 프로젝트 생성, 그리고 프로젝트별 참여 멤버의 권한 및 협업 설정을 수행하는 관리 공간임.

### 2.1. 프로젝트 진입 및 사용자 프로필 전환
* **사용자 프로필 전환 (Top Header)**
  * 우측 상단의 프로필 드롭다운 메뉴를 통해 테스트 및 검토할 가상 사용자를 간편하게 스위칭할 수 있음.
  * 전환 시 해당 사용자의 데이터베이스 권한 등급(Admin 또는 Viewer)이 전역 상태(`appState`)로 자동 주입되며, 해당 사용자가 참여하고 있는 전용 프로젝트 목록만 DB에서 조회해 실시간으로 갱신 로드함.
* **프로젝트 목록 조회 및 진입**
  * **프로젝트 카드 그리드**: 현재 접속한 사용자가 접근 가능한 프로젝트들이 개별 카드로 직관적으로 정렬됨.
  * **카드 정보**: 프로젝트 명칭, 설명글, 업로드된 누적 IFC 모델 개수(`IFC EA`), 그리고 해당 사용자의 프로젝트 내 상세 권한 뱃지(Admin 또는 Viewer)가 표시됨.
  * **Enter 버튼**: 카드 내의 `Enter` 버튼 클릭 시 해당 프로젝트 내부의 3D 모델 뷰어 및 협업 레이아웃 그리드 화면으로 즉시 진입함.

### 2.2. 신규 프로젝트 생성
* **프로젝트 빌더 모달 (New Project 카드)**
  * 대시보드 그리드 끝의 점선으로 둘러싸인 `New Project` 카드 또는 우측 상단의 `New Project` 버튼 클릭 시 프로젝트 생성 팝업이 활성화됨.
  * **프로젝트 기본 정보 기입**: 프로젝트 이름(필수) 및 상세 설명 필드를 입력함.
  * **팀 멤버 초대 (Invite Team Members)**: 시스템에 등록된 전체 사용자 목록 중 초대할 사용자들을 체크박스로 다중 선택하고, 각각의 초기 권한 등급(Viewer 또는 Admin)을 설정할 수 있음.
  * **생성 실행**: 하단의 `Create` 버튼을 누르면 Oracle DB 및 백엔드 서버에 데이터가 등록되며 새 프로젝트 생성이 완료됨.

### 2.3. 프로젝트 멤버 권한 관리 (Project Settings)
* **상세 권한 및 멤버 제어 (Admin 전용)**
  * 프로젝트 카드 중 본인이 `Admin` 권한을 가진 프로젝트의 경우, 카드 우측 상단의 기어 모양 `Settings` 아이콘이 활성화됨.
  * **멤버 권한 수정**: 프로젝트 참여 멤버 목록에서 각 사용자의 보안 권한(Viewer/Admin)을 드롭다운 선택만으로 실시간 즉각 동기화 변경 처리할 수 있음.
  * **참여 권한 회수 (휴지통 아이콘)**: 특정 멤버 우측의 삭제 아이콘을 클릭하여 해당 사용자의 프로젝트 접근 권한을 회수 및 차단함. (본인 권한은 셀프 삭제할 수 없도록 안전 장치 적용).
  * **신규 멤버 추가**: 드롭다운을 통해 아직 프로젝트에 참여하지 않은 다른 시스템 사용자를 선택한 후, 권한 등급을 매핑하여 추가할 수 있음.

## 3. 3D Viewport
* 본 플랫폼의 핵심 시각화 엔진으로, 대용량 IFC/Fragment 모델의 정밀 3D 렌더링, 시각적 조작, 거리 측정, 단면 자르기, 층별 확장 등의 모델 검토 유틸리티를 제공함.

### 3.1. 기본 조작 (CustomCameraControl 기반)
* **화면 이동 (Pan)**: 마우스 휠(가운데 버튼)을 클릭한 상태로 드래그하여 화면을 평행 이동함.
* **카메라 회전 (Orbit)**: `Shift` 키를 누른 상태에서 마우스 휠(가운데 버튼)을 드래그하여 카메라 시점을 기준으로 모델을 회전 탐색함. (마우스 좌/우클릭은 조작 오작동 방지 및 기능 격리를 위해 회전 기능이 할당되어 있지 않음).
* **확대/축소 (Zoom)**: 마우스 휠을 스크롤하여 화면을 확대/축소함. (마우스 포인터가 위치한 지점을 표적으로 정밀 줌인/아웃이 이루어지는 `dollyToCursor` 기능 탑재).
* **더블 클릭 (Focus)**: 3D 모델 상의 객체를 더블 클릭하면 해당 객체를 중심으로 카메라 시점을 자동으로 집중 피팅(Fit to Item)함.
* **근접 시 줌/이동 속도 보정**: 모델 객체에 카메라가 극도로 근접(10.0m 이내)할 때 이동 및 줌 속도가 0에 수렴하는 현상을 막기 위해, 거리에 비례하여 조작 속도를 자동으로 실시간 보정(Boost)해 줌.
* **비행 모드 (Fly Mode)**: 하단 툴바의 비행 아이콘을 클릭하여 활성화하며, 1인칭 시점 탐색이 가능함.
  * **조작법**: 마우스 왼쪽 드래그로 시선 방향을 회전하고, `W` (앞), `A` (왼쪽), `S` (뒤), `D` (오른쪽) 키를 사용하여 시공간 내부를 자유롭게 비행함.
  * **HUD 오버레이**: 비행 모드 진입 시 조준간 오버레이 및 현재 카메라의 실시간 X, Y, Z 절대 좌표가 화면에 표시됨.

### 3.2. 객체 선택 및 박스 선택 (Box Selection)
* **단일 선택**: 3D 공간 상의 객체를 마우스 왼쪽 버튼으로 단클릭하여 하이라이트하고 선택 데이터를 로드함. (카메라 조작 시 오클릭 방지를 위해 마우스 좌클릭은 선택 기능 전용으로 격리됨).
* **다중 선택 (단클릭 조합)**:
  * **Ctrl + 왼쪽 마우스 단클릭**: 기존에 선택된 목록을 유지하면서 클릭한 객체를 다중 선택에 추가함.
  * **Shift + 왼쪽 마우스 단클릭**: 기존 선택 목록 중 클릭한 객체만 선택 해제함.
* **다중 선택 (Box Selection - 드래그)**: 키보드 단축키 조합과 드래그 영역 지정을 통해 여러 부재를 동시에 일괄 선택하거나 제외할 수 있음.
  * **선택 모드 (Ctrl + 마우스 왼쪽 드래그 - 실선 박스)**: 좌측에서 우측으로 드래그하여 박스(파란색 실선 및 투명 채우기)를 그리면, 박스 영역 내에 일부라도 포함(Intersect)되거나 전체가 포함된 객체들이 선택 목록에 일괄 추가됨.
  * **제외 모드 (Shift + 마우스 왼쪽 드래그 - 점선 박스)**: 우측에서 좌측으로 드래그하여 박스(초록색 점선 및 투명 채우기)를 그리면, 기존 선택된 객체들 중 해당 박스 영역에 겹친 부재들이 선택 목록에서 일괄 제거됨.
  * **마우스 커서 피드백**: 박스 드래그 중에는 조준선(Crosshair) 형태로 마우스 커서 스타일이 변경되며, 드래그 종료 시 3D 카메라 조작 권한이 정상 복원됨.

### 3.3. 하단 툴바 (Bottom Toolbar)
* 3D 화면 중앙 하단에 배치된 가로형 도구 모음으로, 단축키와 연계되어 신속한 조작을 지원함.
  * **Show All (단축키 A)**: 숨겨지거나 격리되었던 모든 3D 부재들을 원래 상태로 가시화함.
  * **Ghost Mode (단축키 G)**: 선택되지 않은 배경 객체들을 극도로 높은 투명도(0.01)를 가진 회색 재질로 변환하여, 선택 객체의 맥락은 파악하면서 부하 없이 검토하게 만듦.
  * **Toggle Hidden (단축키 S)**: 시스템에 영구 숨김으로 지정된 객체들의 가시성을 온/오프 토글함.
  * **Focus (단축키 F)**: 카메라를 현재 선택한 객체의 중심 또는 모델 전체 크기에 맞춰 자동 클로즈업 줌인 피팅함.
  * **Hide (단축키 H)**: 선택한 객체들을 화면에서 숨김. (숨겨진 상태에서 한 번 더 실행 시 숨김이 해제됨).
  * **Isolate (단축키 I)**: 선택한 객체들만 남기고 나머지 모든 배경 객체들을 일시적으로 숨김.
  * **Clipper Box (단축키 C)**: 선택한 객체(선택이 없을 시 모델 전체)를 둘러싸는 6면체 단면 상자(Clipping Box)를 생성하여 내부 상세 구조를 단면 검토함.
  * **Colorize (페인트 아이콘)**: 뷰어 화면 상의 객체 표면 색상을 임의의 색상으로 오버라이드하여 시각적으로 분류함.
  * **Floor Exploder (단축키 E)**: 모델을 층(Floor) 단위로 층층이 쪼개어 가로 세로로 분해 배치하는 4D 확장 뷰 기능임. 활성화 시 Y축 분해 반경(Y Scale)을 2배에서 20배까지 제어하는 레이지 슬라이더 바가 동적으로 오픈됨.


### 3.4. 마우스 우클릭 콘텍스트 메뉴 (Context Menu)
* 3D 뷰포트 내부에서 마우스 오른쪽 버튼을 클릭할 때 마우스 포인터 위치에 상황별 메뉴를 띄워 신속한 액션을 지원함.
  * **Items Data**: 현재 선택된 부재들의 속성 데이터를 띄우는 전용 팝업 창을 엶. (테이블 검색 및 JSON 익스포트 연동).
  * **Focus**: 선택 부재로 화면 카메라 조준 피팅.
  * **Hide / Isolate**: 대상 부재 숨기기 또는 격리 탐색.
  * **Clipper Box**: 대상 부재 기준 단면 박스 켜기/끄기.

### 3.5. 좌측 측정 및 단면 도구 바 (Left Utility Toolbar)
* 3D 화면의 좌측 상단에 수직으로 배치된 기능 바로, AI Assistant 호출, 거리 계측 및 수동 단면 조작 기능을 제공함.
  * **AI Assistant (챗봇 아이콘)**: Gemini AI Assistant 패널을 온/오프 토글함. 활성화 시 3D 뷰포트 우측에 사이드바로 열려 자연어로 3D 뷰어 제어 및 모델 데이터 질의를 수행할 수 있음.
  * **Measurements (자 아이콘)**: 3D 공간 상에서 거리를 재는 다중 기능 드롭다운 메뉴를 포함함.
    * **Measure Length (선 자 아이콘)**: 마우스로 연속 클릭한 두 점 사이의 물리적 거리를 실시간으로 측정하여 표시함.
    * **Measure Area (면적 아이콘)**: 마우스 클릭으로 지정한 폐곡선 영역의 평면 면적을 연산함.
    * **Measure Clearance (조준경 아이콘)**: 선택한 두 객체 간의 최단 비행/클리어런스 이격을 자동으로 계산함.
    * **All Clear (지우기 아이콘)**: 생성된 모든 측정 자와 결과 텍스트 라벨을 화면에서 제거함.
  * **Section (단면 아이콘)**: 특정 표면(Face) 기준의 단면 절단을 제어하는 드롭다운 메뉴를 포함함.
    * **Clipper Face**: 3D 부재 표면을 클릭하여 해당 면 방향에 직교하는 절단면(Clipping Plane)을 즉시 생성 및 자유 조정함.
    * **All Clear**: 생성된 모든 수동 단면 자르기 평면을 제거하여 원본으로 복구함.

### 3.6. 뷰어 환경 설정 (Viewer Setting)
* 3D 뷰포트 우측 상단에 배치된 기어 모양 아이콘 버튼을 통해, 뷰어의 시각 물리 파라미터 및 테마를 동적으로 구성함.
  * **Grid 토글**: 3D 공간 밑바닥에 표시되는 물리 그리드(Grid) 격자망의 표시 상태를 온/오프 제어함.
  * **Camera Projection**: 원근감 있는 일반 투시도(`Perspective`) 뷰와 왜곡 없는 정투영 평면도(`Orthographic`) 뷰를 상호 전환함.
  * **Near Plane**: 카메라 렌즈의 전방 한계(Near Plane) 거리를 0.1m에서 20m까지 슬라이더로 조절하여, 전면부 장애물을 뚫고 실내를 검토하는 데 활용함.
  * **Far Plane**: 카메라 렌즈의 후방 시야 거리(Far Plane) 한계를 30m에서 2000m까지 슬라이더로 설정해 모델이 렌더링될 깊이를 정의함.
  * **Color Mode**: 전체 사용자 인터페이스의 테마 컬러를 `Dark`(기본 배경 0x2a2438 적용) 테마 또는 `Light`(배경 0xd6d9dc 적용) 테마로 상호 전환함.

## 4. Viewer 레이아웃 설명
* 뷰어 인터페이스는 3D 뷰포트를 중심으로 화면 좌/우측의 다양한 기능 섹션 패널로 구성되어 직관적인 모델 제어가 가능함.

### 4.1. 섹션 구성
* **3D 중심의 멀티 칼럼 레이아웃**:
  * 뷰어 레이아웃은 중앙의 3D 뷰포트(Viewer)를 기준으로 좌측 칼럼(IFC List, Model Tree)과 우측 칼럼(Selection Data, View Templates), 그리고 필요 시 좌측 도구 바에서 활성화 가능한 우측 사이드바(AI Assistant)로 구성됨.
  * 각 칼럼 및 행 사이의 간격(Gap)을 마우스 드래그하여 각 기능 패널의 크기를 실시간으로 자유롭게 조절(Resize)할 수 있음.
* **각 섹션의 배치**:
  * **좌측 상단**: IFC List (모델 불러오기, DB 저장소 리스트 및 공유 제어)
  * **좌측 하단**: Model Tree (모델의 공간구조 트리 및 엔티티 유형 분류 트리)
  * **중앙**: 3D 뷰포트 (실시간 모델 렌더링 및 3D 마우스 조작)
  * **우측 상단**: Selection Data (선택된 객체의 메타데이터 속성 테이블)
  * **우측 하단**: View Templates (사전 설정된 가시성 템플릿 목록)
  * **우측 사이드바 (토글)**: AI Assistant (자연어 모델 데이터 질의 및 뷰어 제어용 챗봇 패널)

### 4.2. IFC List 섹션으로 모델 DB 저장 및 모델 애플리케이션에 로드하기
* **모델 가져오기 및 DB 저장 (Header Action)**
  * **Import Model 버튼 (`+` 아이콘)**: 로컬의 `.ifc` 파일을 업로드하여 최적화된 `.frag` 형식으로 자동 변환하고, Oracle DB에 저장함과 동시에 모델을 뷰어 화면에 로드함.
  * **Import Model with EDB data 버튼 (상자 안 `+` 아이콘)**: 로컬 `.ifc` 파일을 업로드하고 백엔드 API(`/api/add-edb-data`)를 통해 EDB 데이터를 결합(Enrich)한 후, DB 저장 및 로드를 처리함.
* **Loaded Model (로드된 모델 관리 리스트)**
  * **메모리 로드 현황**: 현재 뷰어 내 3D 공간 상에 로드되어 있는 활성 모델 목록 및 개수를 실시간 표시함.
  * **실시간 검색**: 검색 입력창을 통해 현재 로드된 모델 중 특정 모델을 신속하게 필터링할 수 있음.
  * **가시성 제어**: 모델명 우측의 눈 아이콘(Show/Hide)을 통해 개별 모델의 3D 뷰포트 내 표시 상태를 설정할 수 있음.
  * **메모리 해제**: 휴지통 모양의 Dispose 아이콘 클릭 시 해당 모델을 브라우저 메모리에서 완전히 해제함.
  * **일괄 관리**: 체크박스를 이용해 다수의 모델을 선택한 후 `Select All` 혹은 `Dispose` 버튼으로 일괄 메모리 해제가 가능함.
* **Shared Model (DB 저장 공유 모델 리스트)**
  * **DB 저장소 동기화**: 현재 선택된 프로젝트와 연동되어 데이터베이스(Oracle DB)에 안전하게 영구 저장된 모델 리스트를 관리함.
  * **그룹(Group) 지정 및 필터링**:
    * 개별 모델 단위로 7가지 HSL 색상 그룹(Red, Yellow, Green, Light Blue, Dark Blue, Purple, None) 지정 드롭다운 제공.
    * 리스트 상단에 각 그룹별 할당된 모델 수(EA)가 뱃지로 표시되며, 특정 색상 뱃지 클릭 시 해당 그룹의 모델만 간편하게 필터링할 수 있음.
  * **개별 제어**: 목록 우측 아이콘을 클릭하여 원하는 모델을 즉시 애플리케이션에 로딩(`Load`), 로컬 PC로 파일 다운로드(`Download`), 또는 DB에서 영구 삭제(`Delete`)할 수 있음.
  * **일괄 로드**: 체크박스로 여러 모델을 다중 선택한 후 `Load` 버튼을 통해 한 번에 뷰어로 로드할 수 있음.

### 4.3. Model Tree 섹션으로 모델 공간구조 및 엔티티 구조 확인하기
* **다중 트리 모드 지원 (Header Tab)**
  * **Spatial (공간구조 트리)**: 프로젝트 하위의 Site ➔ Building ➔ Storey(층) ➔ Element(객체) 순서의 물리적·공간적 계층 구조를 트리 구조로 탐색할 수 있음.
  * **Entity (엔티티 트리)**: IFC 표준 클래스 타입(예: IfcWall, IfcSlab, IfcWindow 등)의 엔티티 유형별 계층 구조를 직관적으로 분류하여 보여줌.
* **검색 및 구조 유지 필터링**
  * 실시간 텍스트 검색(`Search...`)창을 통해 트리 내 특정 노드나 요소를 신속하게 필터링하여 식별할 수 있음.
  * 검색 필터 적용 중에도 상위 계층 노드 구조가 손실되지 않고 그대로 유지되어 객체의 공간적 맥락을 정확히 추적할 수 있음.
  * 검색 초기화(`Clear Search`) 버튼을 통해 손쉽게 검색 필터를 해제할 수 있음.
* **트리 제어 및 뷰포트 선택 연동**
  * **일괄 확장/축소**: `Toggle Expanded` 버튼으로 복잡한 트리 계층 구조 전체를 한번에 펼치거나 닫을 수 있음.
  * **Search Selection 기능**: 3D 뷰포트에서 원하는 객체를 마우스로 선택한 후 돋보기 모양의 `Search Selection` 버튼을 누르면, 해당 객체의 이름(Name) 속성을 자동 추출하여 트리 내 해당 노드를 추적하고 자동으로 하위 노드를 확장하여 하이라이트 표시해 줌.

### 4.4. Selection Data 섹션으로 선택된 객체 속성 데이터 확인하기
* **실시간 객체 속성 로드**
  * 3D 뷰포트 상에서 마우스 클릭을 통해 하이라이트된 객체의 IFC 메타데이터 및 상세 속성(Property) 리스트를 동적으로 추출하여 표 형태로 구성함.
  * 여러 개의 객체를 다중 선택할 경우, 실시간 이벤트 연동을 통해 속성이 포함된 객체의 전체 개수가 `Selection Data (N)` 형태로 라벨 뱃지에 동적 표시됨.
* **페이지 분할 (Pagination)**
  * 다수의 대형 객체 정보 동시 조회 시 렌더링 부하를 줄이기 위해 한 페이지당 최대 30개씩 분할하여 데이터를 로드함.
  * 테이블 우측 상단의 페이지 컨트롤러(이전/다음 화살표 버튼 및 `현재페이지 / 전체페이지` 표시)를 통해 대량의 메타데이터를 편리하게 넘겨가며 조회 가능함.
* **속성 데이터 검색 및 내보내기**
  * **텍스트 필터**: 테이블 검색창에 단어를 입력하여 복잡한 속성 키(Key)나 값(Value)을 즉시 필터링하여 찾을 수 있음.
  * **데이터 내보내기 (`Export Data` 아이콘)**: 현재 화면에 표시 및 선택된 객체들의 메타데이터와 속성 정보를 로컬 PC에 `.json` 파일(`ElementData.json`)로 즉시 내보내기(Export)할 수 있어 타 솔루션 및 보고서 활용에 용이함.

### 4.5. View Templates 섹션으로 미리 설정한 가시성 활용하기
* **사전 설정된 가시성 관리**
  * 모델 분석, 공정별 검토 등 목적에 맞게 미리 구성해 둔 뷰 템플릿(Visibility, 색상 하이라이트 오버라이드 값 등) 목록을 한눈에 관리함.
* **템플릿 즉시 적용 및 초기화**
  * **Apply Template 버튼 (스포이트 아이콘)**: 템플릿 행의 드로퍼 단추 클릭 시, 해당 뷰 템플릿의 가시성 색상 옵션이 3D 모델 뷰포트에 실시간으로 매핑되어 특수 가시성 상태로 즉각 전환됨.
  * **Reset View 버튼 (눈 아이콘)**: 적용되어 있는 가시성 템플릿 효과를 완전히 비활성화하고, 모델의 원본 3D 그래픽 표현 상태(디폴트 뷰)로 화면을 신속하게 초기화함.

## 5. BCF Manager 레이아웃
* 본 플랫폼은 OpenBIM의 핵심 개방형 표준 중 하나인 BCF(BIM Collaboration Format) 표준 규격을 탑재하여, 3D 모델의 이슈 발견부터 조치 완료까지 전체 협업 이력을 디지털 데이터로 보존하고 동기화할 수 있는 BCF Manager 레이아웃을 제공함.

### 5.1. 섹션 구성
* **상하 2단 분할 및 다중 패널 배치**:
  * BCF Manager 레이아웃은 화면 상단의 3분할 영역과 화면 하단의 전체 가로 영역으로 구성됨.
  * **상단 좌측**: IFC List (모델 업로드 및 활성 모델 제어 패널)
  * **상단 중앙**: 3D 뷰포트 (이슈 관련 객체 및 카메라 뷰포인트 실시간 시각화)
  * **상단 우측**: BCF List (DB 저장된 BCF 파일 검색, 임포트, 다운로드 및 삭제 패널)
  * **하단 전체**: Topic List (이슈 리스트 조회 및 새 토픽 생성/기존 토픽 수정 폼 패널)
* **동적 폼 스왑 (Dynamic Panel Switch)**:
  * Topic List 상단의 `Create Topic` 실행 시, 하단 우측의 기존 패널 공간이 **New Topic 작성 폼**으로 실시간 전환됨.
  * 기존 토픽의 수정/상세 편집을 트리거할 경우, 해당 영역이 **Update Topic 폼 및 댓글(Comments) 사이드바**로 동적으로 스왑됨.
* **레이아웃 전환 시 자동 이벤트**:
  * 사용자가 BCFManager 레이아웃으로 화면을 전환할 때, DB로부터 최신 BCF 파일 및 토픽 정보를 자동으로 동기화하여 가져오는 새로고침(Auto-Refresh) 이벤트가 실행됨.

### 5.2. BCF 개략 설명
* **개방형 BIM 협업 표준**: BCF는 IFC 원본 모델 파일을 직접 수정하는 무겁고 위험한 작업 방식 대신, 이슈가 발생한 부분의 '3D 카메라 뷰포인트 정보', '선택 및 하이라이트된 객체 목록', '현장 스냅샷', '문제 분류 및 상태', 그리고 '댓글'을 가벼운 XML 기반 패키지 파일(`.bcf`)로 상호 교환할 수 있게 설계된 빌딩스마트(buildingSMART) 공식 표준임.
* **프로젝트 의사소통 가속화**: 뷰어를 통해 설계 오류, 간섭 구간, 현장 변경 요구사항 등을 정확히 짚어내고, 카메라 위치와 시각적 상태를 그대로 원격 협업자에게 공유하여 소통 비용을 획기적으로 절감함.

### 5.3. BCF - Topic - Viewpoint - Comment 로 이어지는 구조 설명
* **BCF (최상위 컨테이너)**: 복수의 협업 이슈 사항들을 하나로 묶어 관리 및 교환할 수 있는 최상위 패키지이자 데이터 저장 파일 단위임.
* **Topic (토픽/이슈 건)**: BCF 내에 등록되는 개별 이슈 항목을 뜻함. 제목(Title), 상세 설명(Description), 상태(Status - Open/Assigned/Closed/Resolved), 타입(Type), 우선순위(Priority), 생성자 및 생성일자, 담당자(Assignee), 만료기한(Due Date), 분류 태그(Labels), 스테이지(Stage) 등의 협업용 정보 필드로 구성됨.
* **Viewpoint (뷰포인트/카메라 뷰)**: 토픽이나 댓글이 생성되던 순간의 3D 공간 상 카메라 좌표(x, y, z), 시선 방향(Direction), 업 벡터(Up Vector), 단면 상자(Clipping Box)의 설정, 하이라이트된 객체의 Express ID 등 시각적 3D 뷰의 캡처 정보 및 Base64 인코딩 스냅샷 이미지 데이터를 의미함.
* **Comment (댓글/이력)**: 해당 토픽을 해결하기 위해 협업자 간에 주고받은 텍스트 의견 및 조치 이력임. 댓글마다 별도의 신규 Viewpoint를 연계할 수 있어 시간에 따른 3D 뷰의 상태 변화를 시각적으로 추적 가능함.

### 5.4. BCF List 섹션 설명
* **모델 연계형 BCF 자동 검색**: 현재 3D 뷰어에 로드된 IFC 모델들과 직접적으로 연결된 BCF 파일들만 필터링하여 데이터베이스에서 추출하고 표시해 줌.
* **상단 도구 모음**
  * **실시간 검색**: 입력창을 통해 파일명을 기준으로 관련 BCF 리스트를 빠르게 실시간 필터링할 수 있음.
  * **Import BCF 버튼 (`+` 아이콘)**: 로컬 PC의 `.bcf` 파일을 읽어와 Oracle DB에 저장함. (이슈 작성 및 편집 상태인 경우에는 안전을 위해 작동이 제한됨).
  * **Refresh 버튼 (새로고침 아이콘)**: DB에서 최신 BCF 목록을 다시 읽어와 갱신함.
* **목록 행 기능**
  * **연계 모델 조회 (모델 아이콘)**: 마우스 오버 시 해당 BCF 파일이 참조하고 있는 연계 IFC 모델 명칭들을 툴팁으로 제공함.
  * **Load Topics 버튼 (`import` 아이콘)**: 선택한 BCF 파일 속 모든 토픽 정보를 로드하여 아래의 **Topic List** 패널에 뿌려줌.
  * **Download BCF 버튼 (다운로드 아이콘)**: DB의 BCF 데이터를 패키징하여 로컬 PC에 `.bcf` 파일로 내보냄.
  * **Delete BCF 버튼 (휴지통 아이콘)**: 확인창을 거친 후 BCF 데이터를 Oracle DB에서 영구 삭제함.

### 5.5. Topic List 섹션 설명 (각 버튼들 기능)
* 로드된 토픽들의 개요 및 라이프사이클을 추적하고 다채로운 분석 툴을 조작하는 공간임.
* **섹션 타이틀 상태 요약**: 섹션 라벨 우측에 실시간으로 `Total = Open + Assigned + Closed + Resolved` 개수를 분류하여 요약 표시함.
* **상단 액션 버튼 그룹 (좌측)**
  * **Create Topic**: 새로운 토픽을 추가할 수 있는 신규 작성 폼(New Topic) 패널로 화면을 전환함.
  * **Markers**: 토픽들에 기록된 간섭 지점(Clash Point) 좌표들에 빨간 마커(Marker)들을 3D 뷰포트 내에 시각적으로 온/오프 표시함.
  * **Clear**: 현재 화면의 Topic List에 활성화된 모든 토픽 데이터를 비움.
  * **Save BCF**: 현재의 토픽 추가, 수정, 삭제 사항을 뷰어 DB에 저장 및 동기화함.
  * **Send to TDVS**: 5.8절에 설명된 대로 현재 로컬 토픽들을 외부 TDVS 시스템으로 전송 및 매핑 처리함.
  * **Sync TDVS (강조 활성화 기능)**: TDVS 외부 시스템의 최신 토픽 정보 및 추가된 의견(Comment)들을 현재 로컬 뷰어로 역동기화하여 가져옴. (동기화할 항목이 있는 경우 강조 색상으로 활성화됨).
* **상단 액션 버튼 그룹 (우측 검색 및 유틸리티)**
  * **실시간 검색**: 제목과 설명을 기준으로 토픽 리스트를 실시간 검색함.
  * **지우기 버튼 (Clear Search)**: 검색 필터를 초기화함.
  * **제외 버튼 (Exclude Search)**: 현재 검색 결과에 나타난 토픽들을 목록에서 일괄 삭제함.
  * **격리 버튼 (Isolate Search)**: 검색 결과에 부합하는 토픽들만 남기고 나머지를 목록에서 일괄 비움.
  * **Pagination (페이징)**: 목록 하위의 페이징 템플릿을 통해 한 페이지당 30개씩 분할 탐색을 지원함.
* **토픽 테이블 및 편집 트리거**
  * 토픽 행을 마우스 더블클릭하거나 우측 연필(Edit) 단추를 누르면 해당 토픽의 **Update Topic** 수정 패널이 열리며, 휴지통(Delete)을 누르면 토픽이 삭제됨.

### 5.6. 새로운 토픽 만들기 설명
* **New Topic 패널**을 사용하여 신규 협업 이슈를 구성하고 등록함.
* **스냅샷 캡처 및 이미지 로드 (필수)**
  * 새로운 토픽 생성을 위해서는 반드시 3D 뷰 캡처나 관련 스냅샷 이미지가 지정되어야 함.
  * **Capture 버튼 (카메라 아이콘)**: 현재 3D 뷰포트의 카메라 상태(줌, 각도, 선택 객체, 단면 등)를 캡처하여 좌측 썸네일로 즉시 자동 빌드함.
  * **Import 버튼 (파일 아이콘)**: 로컬에 보관 중인 현장 사진 등 외부 이미지(`PNG`, `JPG`) 파일을 썸네일 스냅샷으로 직접 연동함.
  * **Restore 버튼 (조준경 아이콘)**: 이미 캡처된 뷰포인트를 3D 화면에 투사해 사전 확인해 볼 수 있음.
* **이슈 데이터 정의**
  * **이슈명 (Title)**: 필수 입력 필드이며 미입력 시 하단의 Add Topic 등록 버튼이 비활성화됨.
  * **설명 (Description)**: 발견된 이슈의 구체적인 내용 및 조치 요구 사항을 기재함.
  * **이슈 분류 및 속성**: Type, Status, Assignee, Priority, Labels(복수 태그 지정 및 색상 뱃지 반영), Due Date, Stage 속성 드롭다운을 통해 세부 카테고리를 설정함. (Author 항목은 시스템 관리자(Admin) 권한 보유 시에만 편집 가능함).
* **등록 절차**: 작성 후 하단의 `Add Topic` 버튼을 누르면 목록에 우선 등록되며, 이를 타 사용자에게 전파하려면 최종적으로 메인 패널의 `Save BCF`를 실행하여 DB에 연동해주어야 함.

### 5.7. 토픽 업데이트 하기 설명
* **Update Topic 패널**을 통해 기존에 등록되어 검토 중인 토픽의 상세 속성을 변경하거나 댓글 이력을 기록함.
* **보호 및 롤백(Rollback) 기능**
  * 수정 패널이 활성화되는 시점에 토픽의 댓글 목록, viewpoints, 원본 카메라 좌표 등의 데이터가 백업 메모리에 기록됨.
  * 수정 조작 중에 우측 하단의 `Cancel` 버튼을 클릭하면, 작업 중인 모든 내역을 무시하고 백업된 데이터로 즉시 복원(Rollback)하여 안전하게 목록 뷰로 돌아감.
* **뷰포인트 업데이트 및 스냅샷 변경**
  * 기존 뷰포인트를 현재 뷰포트 상태로 갱신하고 싶을 때 `Capture` 버튼을 클릭하거나, `Import` 버튼을 눌러 새 스냅샷 이미지로 손쉽게 덮어쓸 수 있음.
  * `Restore` 버튼을 이용해 기존 뷰포인트의 3D 공간 상 상세 각도 및 객체 상태를 3D 뷰포트에 역으로 투영해 볼 수 있음.
* **댓글(Comments) 히스토리 관리**
  * 우측 상단의 화살표 버튼을 클릭해 댓글 사이드바를 활성화(Expand)하거나 접어둘 수 있음.
  * 댓글창 내에서 협업 논의 텍스트를 기입하고 신규 뷰포인트를 붙여 업로드할 수 있어, 시계열 흐름에 따른 보완 상태와 이슈 대화 내용을 상세하게 관리함.
* **업데이트 적용**: 수정 완료 후 `Update Topic` 버튼을 클릭하여 토픽 목록에 반영하며, 최종적으로 `Save BCF` 버튼을 실행해 Oracle DB에 저장함.

### 5.8. TDVS 에 토픽 보내기, 동기화시키기 기능 설명
* **TDVS 데이터 연계 환경**: 플랫폼은 외부 사업 및 마일스톤 관리를 수행하는 TDVS 데이터베이스 테이블(`SI_BCF_TOPIC`, `SI_BCF_COMMENT`)과 전용 API를 통해 상호 연결됨.
* **Send to TDVS (토픽 외부 전송)**
  * 현재 로드된 BCF 토픽들과 하위 댓글 내역 전체를 취합하여 TDVS 서버 API(`/api/bcf/send-to-tdvs`)로 전송함.
  * 전송 시 3D 카메라 좌표 정보를 TDVS 공간 체체에 적합하도록 Z-up 좌표계(`X=x, Y=-z, Z=y`)로 자동 매핑 변환함.
  * 연계된 IFC 파일 이름(`priFile`)을 자동 추출하여 TDVS 레코드와 매핑시킴.
  * 전송 완료 시, 외부 시스템에서 발급한 관리 번호인 **mrimsNo**가 로컬 BCF 토픽 데이터에 자동으로 연동 및 각인됨.
* **Sync TDVS (실시간 역동기화)**
  * **변경 감지 및 강조 표시**: 현재 로드된 활성 IFC 모델명들을 조회해 TDVS에 등록된 BCF 정보를 실시간 검색하고, 로컬에 누락된 새 토픽이 있거나 기존 토픽 내에 로컬 뷰어에 반영되지 않은 TDVS 상의 추가 댓글(최종 승인된 댓글 인덱스 `ackCommentNo` 필터링 기준)이 존재하는 경우, `Sync TDVS` 버튼을 **강조색(Accent Color)으로 강조 활성화**하여 사용자에게 변경 사실을 인지시킴.
  * **동기화 적용**: 활성화된 `Sync TDVS` 버튼 클릭 시 외부 TDVS 서버로부터 최신 토픽 데이터와 댓글 리스트를 파싱 및 빌드하여, 로컬 BCF 토픽 컬렉션 내에 누락 없이 역동기화(Sync)하고 병합함.
* **동기화 상태 인디케이터 (Action 컬럼 아이콘)**
  * 토픽 목록 테이블의 Action 컬럼 좌측에 동기화 상태 아이콘이 실시간 표시됨.
  * **체크 표시 아이콘 (`Synced with TDVS`)**: 해당 토픽의 모든 정보가 외부 TDVS와 일치하여 최신 상태로 유지되고 있음을 의미함.
  * **경고 표시 아이콘 (`Unsynced comments available...`)**: 해당 토픽 하위에 동기화할 수 있는 새로운 외부 댓글 정보가 TDVS 상에 존재함을 사용자에게 경고함. 상세 편집 패널을 열어 정밀 동기화를 수행해야 함을 안내함.
* **코멘트(Comment) 세부 동기화 제어**
  * 토픽 상세(Update Topic) 패널의 Comments 탭 내부 뷰포인트 카드 우측에 **파란색 새로고침 아이콘 버튼**이 활성화됨.
  * 이 버튼은 해당 뷰포인트에 매칭된 TDVS 상의 미동기화 댓글 개수를 파악하여 나타나며, 클릭 시 해당 뷰포인트 그룹 내의 TDVS 새 댓글들만 정밀하게 로컬 BCF 토픽으로 로드(Import)함.
* **사용자 수동 다듬기(텍스트 가공) 및 Acknowledge Sync 활용**
  * **텍스트 다듬기 필요성**: TDVS 서버로부터 가져온 댓글들은 여러 코멘트가 특수 구분자(`;;`)로 묶여 있거나 기계적으로 병합되어 있어 가독성이 떨어질 수 있으므로, 사용자가 3D 화면 상황을 확인하면서 문맥에 맞춰 가독성 있게 텍스트를 수정·편집하는 수동 정리가 권장됨.
  * **Acknowledge Sync (동기화 확인/완료)**: 수동 다듬기가 끝난 후 댓글 창 좌측 하단의 `Acknowledge Sync` 버튼을 클릭하면 외부 TDVS 데이터베이스에 동기화 최종 완료 플래그를 전송함.
  * 동기화 완료 처리 후, 메인 토픽 테이블 목록의 경고 표시 아이콘이 일반 체크 표시 아이콘(`Synced / Acknowledged`)으로 변경되어 정상 상태로 복원됨.

## 6. ClashDetection 레이아웃 설명
* 본 플랫폼은 외부 솔루션 없이 웹 브라우저 자체적으로 3D 객체 간 물리적 교차 및 이격 거리를 정밀 분석하는 클라이언트 측 간섭 검토(Clash Detection) 기능 패널을 제공함.
* Web Worker를 활용한 백그라운드 병렬 연산을 지원하여, 수십만 개의 부재를 검토할 때도 UI가 멈추지 않고 신속하게 분석 결과를 시각화함.

### 6.1. 섹션 구성
* **상하 2단 분할형 검토 환경**:
  * Clash Detection 레이아웃은 화면 상단의 3D 뷰 중심 3분할 레이아웃과 하단부의 대형 간섭 리스트 영역으로 분할됨.
  * **상단 좌측**: IFC List (간섭 검토 대상 모델의 로드 및 표시 제어)
  * **상단 중앙**: 3D 뷰포트 (검출된 간섭점 마커 및 간섭 객체 하이라이트 투영)
  * **상단 우측**: Selection Data (선택한 간섭 부재의 상세 속성 데이터 확인)
  * **하단 전체**: Clash List (간섭 모드 설정, Tolerance 제어, 간섭 검출 병렬 실행 및 결과 리스트 테이블)
* **간격 조절 지원**:
  * 마우스 드래그를 통해 상하 row의 크기 및 좌우 column의 비율을 유연하게 조정하여, 간섭 테이블 데이터를 넓은 화면에서 쾌적하게 분석할 수 있음.

### 6.2. Clash List 상부 버튼들 기능 설명
* **Clash Mode Dropdown (Hard / Soft)**
  * **Hard Clash**: 물리적으로 부재들의 형상이 공간 상에서 겹치는 일반적인 간섭을 검출함. 선택 시 수치 입력창의 라벨이 `Tolerance (m)`로 전환됨.
  * **Soft Clash**: 부재 간 안전 거리 또는 유지보수를 위해 반드시 확보해야 할 최소 이격 거리를 검출함. 선택 시 수치 입력창의 라벨이 `Clearance (m)`로 전환됨.
* **수치 입력창 (Tolerance / Clearance (m))**
  * 간섭 오차 한계(Tolerance, Hard Clash 시 기본 0.05m) 또는 요구 이격 거리(Clearance, Soft Clash 시 기본 0.05m)를 미터 단위로 설정함.
* **Run Clash 버튼 (번개 아이콘)**
  * **조건별 검토**: 3D 뷰포트 상에서 임의의 부재들을 선택(Selection)한 후 클릭 시, 선택된 객체들(Set A)과 전체 활성 모델 부재들(Set B) 간의 간섭만 타겟팅하여 빠르게 계산함. 선택이 없는 상태에서 클릭하면 전체 활성 모델 부재 간 간섭을 일괄 검출함.
  * 검토 완료 시, 검출된 간섭 개수 정보가 팝업되며 Clash Detection 패널 활성화 상태로 자동 포커싱됨.
* **Markers 버튼 (지도 아이콘)**
  * 3D 뷰포트 공간 상의 모든 간섭 지점 좌표에 구형의 간섭 마커(Clash Marker)를 시각적으로 투영함.
  * 마커의 색상은 간섭의 Badge 상태값(New: 빨강, Hold: 노랑, Exclude: 파랑)과 실시간 연동되어 렌더링됨.
* **Delete 버튼 (휴지통 아이콘)**
  * 테이블 리스트에서 체크박스로 간섭 또는 간섭 그룹 행을 선택한 경우 활성화되며, 선택된 간섭 이력을 결과 목록에서 제거함.
* **Clear 버튼 (지우기 아이콘)**
  * 현재의 간섭 검사 데이터를 메모리에서 완전 소거하고, 3D 화면 상의 모든 마커 및 특수 하이라이트를 리셋하여 뷰어 상태를 원본으로 복원함.
* **To Topic 버튼 (디스크 아이콘)**
  * 현재 리스트에서 Badge가 **"New"**인 검토 항목들을 일괄 3D 뷰포인트 좌표와 연계하여 BCF Topic으로 일괄 자동 생성하고 목록에 등록함.
  * "Hold" 및 "Exclude" 상태 of 간섭 항목들은 Oracle DB 연동 스토리지(`/api/clash-manager/upsert`)에 기록하여, 향후 모델 버전이 새로 개정(Revision)되더라도 해당 간섭 필터 상태가 손실 없이 유지되도록 설계됨.
* **Export 버튼 (내보내기 아이콘)**
  * 현재 분석 필터가 반영된 간섭 검출 내역 리스트 전체(그룹 행 제외)를 CSV 파일(`clash_list.csv`)로 내보내 로컬 다운로드함.
* **Badge Filter Dropdown (All / New / Hold / Exclude)**
  * 간섭 검토 상태(Badge) 값에 의거하여 테이블 목록의 표시 대상을 선택적으로 필터링함.
* **Group Dist. (m) 수치 입력창**
  * 여러 개의 오밀조밀한 간섭 점들을 하나의 그룹으로 묶을 거리(m) 반경을 제어함. 값을 변동하면 실시간 계산을 거쳐 간섭 테이블의 그룹 묶음 크기가 즉시 재조정됨.
* **실시간 검색창 및 검색 제어**
  * `Search...` 입력창에 특정 단어를 기입하여 연계된 모델명이나 IFC 엔티티 명칭을 필터링함. (필터링 중에도 트리 상하위 계층 보존).
  * 검색 초기화(지우기), 검색 대상 삭제(제외), 검색 대상 외 소거(격리) 유틸리티 버튼을 함께 배포함.
* **Pagination (페이징)**
  * 대용량 분석 결과를 효율적으로 탐색할 수 있도록 페이지당 30개 단위로 페이지 번호를 매겨 출력을 제어함.

### 6.3. Clash List 각 컬럼 및 Action 버튼들 설명
* **Clash List 테이블 컬럼 정의**
  * **Badge**: 검토 작업 현황에 맞춰 `New`(신규 검토 필요), `Hold`(검토 보류), `Exclude`(정상 시공성 확보 등 간섭 예외) 상태를 변경할 수 있는 전용 드롭다운임.
  * **Model1 / Model2**: 간섭을 발생시킨 대상 부재들의 각각 소속된 모델명.
  * **Entity1 / Entity2**: 간섭 부재들의 IFC 클래스 타입명 (예: IfcWall, IfcPipeSegment 등).
  * **Object1 / Object2**: 간섭 부재들의 고유 이름(Name) 정보.
    * 여러 간섭이 가까워 하나로 합쳐진 **그룹(Group) 행**의 경우, 해당 그룹에 포함된 총 부재 수와 간섭 횟수가 표기됨 (예: `8 objects, 4 clashes`).
  * **Position**: 3D 공간 상에서 간섭이 감지된 곳의 대표 절대 X, Y, Z 좌표.
* **목록 행 클릭 동작 (Go to Clash)**
  * 간섭 목록에서 특정 행을 클릭하면 해당 행이 단일 강조되며, 3D 뷰포트 상에서 **간섭 부재가 겹치는 위치로 카메라가 즉시 자동 이동하고 클로즈업 뷰를 제공**함. (그룹 행인 경우 그룹 중심점으로 카메라 이동 및 그룹 부재 전체 가시화).
* **Action 컬럼 내 개별 제어 버튼**
  * **Select Objects 버튼 (지정 아이콘)**: 간섭을 일으킨 두 부재(그룹 행의 경우 그룹 전체 대상 부재들)를 3D 뷰포트 상에 직접 선택(Highlight) 상태로 자동 매핑함.
  * **Delete Clash/Group 버튼 (휴지통 아이콘)**: 해당 간섭 정보 한 건(혹은 해당 그룹 전체)을 결과 테이블에서 즉시 제외 처리함.
  * **Create Topic/Create Topic for Group 버튼 (디스크 아이콘)**: 해당 건 또는 그룹 전체를 3D 카메라 뷰포인트 정보를 포함한 개별 BCF Topic으로 신속하게 등록할 수 있도록 이슈 작성 화면으로 자동 연결함.

## 7. Queries 및 Dashboard 레이아웃 설명
* 본 플랫폼은 대량의 IFC 객체 정보 속에서 필요한 부재들만 다차원 조건으로 필터링하고 검색하는 기능 및 전체 모델 구성을 시각적 차트로 파악할 수 있는 통계 기능 패널을 제공함.

### 7.1. 섹션 구성
* **통계 차트와 조건 검색의 통합 3칼럼 구성**:
  * 3D 뷰포트를 중앙에 두고 좌측에는 정밀 조건 쿼리 빌더가, 우측에는 속성 데이터 조회 패널이, 중앙 하단에는 통계 대시보드가 조화롭게 결합된 구성임.
  * **좌측 전체 (Full Height)**: Queries (Query Builder 폼 및 Saved Queries 테이블 목록)
  * **중앙 상단**: 3D 뷰포트 (검색/차트 조각 클릭으로 필터링된 객체의 실시간 시각적 하이라이트)
  * **중앙 하단**: Dashboard (카테고리별 막대 차트 및 중첩 도넛 차트 배치)
  * **우측 전체 (Full Height)**: Selection Data (선택된 객체들의 메타데이터 목록 테이블)
* **상하/좌우 드래그 리사이징**:
  * 칼럼 간의 너비 및 중앙부 3D 뷰포트와 통계 대시보드 간의 높이를 자유롭게 조정하여 작업 화면을 최적화할 수 있음.

### 7.2. Queries 섹션 - Query Builder 기능 설명 및 작성법, Saved Queries
* **Query Builder (쿼리 빌더) 기능 및 작성법**
  * 사용자가 지정한 조건들을 복합적으로 조합하여 모델 상의 부재들을 일치 테스트하는 동적 검색기임.
  * **조건 입력란 작성 가이드**:
    * **Query Name**: 생성할 쿼리의 이름을 입력하는 필수 필드임.
    * **Entity**: IFC 클래스 유형(예: WALL, SLAB 등)을 기입함. (대소문자 무관 및 정규식 매칭 처리).
    * **Attribute Name / Attribute Value**: IFC 고유 속성 항목명(Name 등)과 해당 속성이 품고 있을 목표 값을 필터링함.
    * **PropertySet Name**: 찾고자 하는 속성 세트(Pset_xxxx)의 범위를 지정함.
    * **Property Name / Property Value**: 지정한 속성 세트 하위의 속성 명칭과 구체적인 값 조건을 필터링함. (참/거짓(True/False) 형태의 Boolean 속성은 시스템 자체 판정으로 t/f, 1/0 등도 유연하게 해석함).
    * **Container Entity / Container Name**: 부재가 내장된 IFC 공간 구조(예: STOREY ➔ 1F 등)를 기반으로 필터링함.
  * **쿼리 생성 및 선택 피드백**: 조건을 다 기입하고 하단의 `Create Query` 단추를 누르면 쿼리 목록에 등록됨과 동시에, 조건을 완벽히 만족하는 대상 객체들이 **3D 뷰포트 상에 즉각 강조 하이라이트(Select)**되어 나타남.
  * **Clear**: 쿼리 빌더의 모든 입력 양식 데이터를 한꺼번에 비우고 초기화함.
* **Saved Queries (저장된 쿼리 목록)**
  * 현재 저장되어 즉시 사용 가능한 쿼리 리스트를 출력함.
  * 목록 상단의 검색 필터를 이용해 원하는 이름의 쿼리를 검색할 수 있음.
  * 저장된 쿼리 테이블 행을 마우스로 클릭하면, 해당 쿼리 탐색 조건이 모델에 실시간 적용되어 매칭 부재들이 뷰포트 공간에 바로 하이라이트 투영됨.
  * **행별 액션 버튼**:
    * **Select items (지정 아이콘)**: 해당 쿼리의 검색 조건에 만족하는 객체들을 3D 뷰포트 상에 즉각 강조 하이라이트(선택) 처리함.
    * **Load to Query Builder (가져오기 아이콘)**: 저장된 쿼리의 상세 검색 조건 데이터를 Query Builder 입력 필드로 자동 주입하여, 기존 쿼리를 손쉽게 재편집하거나 재조정할 수 있도록 지원함.

### 7.3. Dashboard 섹션 - 차트 설명, 차트 조각 클릭시 뷰어 객체 선택 연동 기능 설명
* **Dashboard 통계 분석 기능**
  * 모델 내에 로드된 부재들의 정보를 집계하여 그래프 및 다중 링 차트로 도식화한 시각적 브리핑 화면임.
* **상부 유틸리티 버튼 (Refresh)**
  * 새로고침 아이콘 단추를 클릭해 실시간으로 모델 데이터 변경 사항 및 분류 내역을 계산하여 차트에 새로 고침 반영함.
* **차트 구성 설명**
  * **By Category (카테고리별 막대 차트)**:
    * 가장 수량이 많은 상위 6개 IFC 카테고리(예: Wall, Slab, Pipe 등)를 막대그래프로 표출함.
    * 막대 상단부에 해당 카테고리의 실제 부재 수 수치 라벨을 표시함.
    * Purple부터 Green까지 점진적으로 변하는 HSL Hues 그라데이션 컬러로 조화롭게 렌더링됨.
  * **By Category & Type (중첩 도넛 차트 - Nested Doughnut)**:
    * 상위 6개 카테고리 각각의 상세 타입 분포 비율을 이중 링 형태로 중첩 도식화한 차트임.
    * **안쪽 링(Inner Ring)**: PredefinedType(IFC 표준 정의 타입)별 총 수량 비중을 표현하며, 카테고리의 대표 톤을 기초로 층층이 밝기(Lightness)를 조정한 그라데이션으로 차별화함.
    * **바깥쪽 링(Outer Ring)**: ObjectType(사용자 정의 물리 타입)별 상세 점유 비율을 나타내며, 수량의 밀도에 맞춰 점진적 투명도(Alpha)를 적용해 타입 분포를 가시화함.
    * 도넛 중앙에는 해당 카테고리의 전체 누적 개수가 강조 수치 라벨로 영구 표기됨.
* **차트 조각 클릭 시 뷰어 객체 연동 (Ghost & Focus)**
  * 차트 내부의 특정 막대 기둥이나 도넛 링 조각(Slice)을 클릭하면 3D 뷰어와 연계되어 정밀 시각 피드백을 수행함.
  * **Ghost (반투명화) 효과**: 클릭하여 선택한 특정 유형/카테고리에 해당하지 않는 나머지 배경 모델 부재들을 전부 반투명(Transparent) 재질로 변화시켜 시각적 방해 요소를 차단함.
  * **Focus & Highlight (초점 이동 및 하이라이트)**: 해당 차트 조건에 완벽히 매칭되는 객체들만 3D 공간 상에서 선명한 선택 색상으로 하이라이트 처리함과 동시에, **선택된 대상들이 화면에 꽉 차도록 카메라를 자동 핏(Fit to Items) 및 시선 자동 정렬**시킴.
  * 빈 바탕 영역을 클릭하거나 이미 클릭했던 영역을 재선택하면 모든 반투명 재질이 투명도 원래 상태로 복원되며 하이라이트가 복구 해제됨.

## 8. IDS Check 레이아웃 설명
* 본 플랫폼은 IFC BIM 모델의 품질 및 납품 데이터 무결성을 검증하기 위한 국제 표준인 IDS(Information Delivery Specification)를 준수하여, 모델 정보의 규격 정합성을 자동으로 검증하는 검사 패널을 제공함.

### 8.1. 섹션 구성
* **3분할 가로 단일 레이아웃**:
  * IDS 사양 정의 및 실시간 검증, 합불 판정 결과 조회를 가로 3분할된 레이아웃에서 원스톱으로 처리할 수 있는 구조임.
  * **좌측 칼럼**: IDS Specs 패널 (아코디언 탭 구조로 구성됨)
    * **Spec. List**: 사전에 정의된 규격 리스트
    * **Spec. Builder**: 임의의 검증 룰 생성 입력 창
    * **Spec. Check Results**: 스펙 검증 실행에 따른 객체별 Pass/Fail 판정 결과 리스트
  * **중앙 칼럼**: 3D 뷰포트 (검사 결과 실패한 부재들의 다크 고스트 및 빨간색 강조 3D 뷰 연동)
  * **우측 칼럼**: Selection Data (선택된 검증 대상 부재의 메타데이터 및 상세 속성 목록 확인)
* **아코디언 토글 제어**:
  * 좌측 칼럼의 각 하부 패널은 아코디언 타이틀을 클릭하여 접거나 펼쳐서 사용 가능하며, 칼럼 경계를 드래그하여 패널 너비를 최적화할 수 있음.

### 8.2. IDS 설명
* **데이터 품질 검증 표준**: IDS는 빌딩스마트(buildingSMART)의 공식 표준 사양으로, IFC 모델 납품 시 요구되는 필수 부재 유형별로 속성 세트(Pset), 물량 세트(Qto), 직속 속성(Attribute)이 누락 없이 규정된 서식과 값 패턴대로 채워져 있는지 머신러닝/자동 연산 형태로 검사 및 판정하는 규격임.
* **오류 원천 차단**: 시공 검토, 공정 시뮬레이션, 물량 산출 전 단계에서 데이터 유효성을 선제적으로 자동 체크하여 정보의 신뢰성을 확보함.

### 8.3. Spec. List (검사 기준 목록)
* 모델 분석에 사용할 사전 정의된 검사 사양(Specification) 및 사용자가 새롭게 빌드하여 추가한 규칙 목록을 한눈에 관리하고 탐색하는 영역임.
* **구성과 동작**:
  * 각 행마다 스펙의 고유 명칭(`Name`)과 적용되는 상세 검증 조건(`Description`)이 출력됨.
  * **Duplicate GUIDs (GUID 중복 검사)**: 모델 내의 모든 고유 Global ID 중복 생성 여부를 체크하는 기본 스펙이 내장되어 있어, 모델의 무결성과 개별 부재의 고유성을 우선적으로 검사할 수 있음.
  * **검사 실행 (플레이 버튼)**: 우측의 플레이 아이콘을 클릭하면 즉시 활성화된 모든 모델을 탐색하여 해당 규칙에 맞는지 검증 테스트를 일괄 구동함.

### 8.4. Spec. Builder (사양 규칙 빌더)
* 사용자가 비즈니스 시나리오 및 납품 기준에 적합한 신규 속성 검증 규격 조건을 직접 구성하고 즉시 구동 및 리스트에 반영하는 도구 모음임.
* **규격 조건 생성 입력**:
  * **Entity**: 사양 검사의 타겟이 되는 IFC 부재 유형(예: WALL, SLAB, PIPESegment 등)을 대소문자 구분 없이 입력함 (정규식 부분 패턴 포함 감지 적용).
  * **Requirement Type (Property / Quantity / Attribute)**:
    * **Property**: Pset 속성 세트 하위의 속성 명칭과 값을 검증할 때 선택함.
    * **Quantity**: Qto 물량 세트 하위의 수치 값 유무를 검증할 때 선택함.
    * **Attribute**: IFC Entity 본체의 고유 데이터 항목(Name, GlobalId 등)을 검증할 때 선택함.
  * **Pset / Qto 명칭**: Property 또는 Quantity 선택 시 입력창이 활성화되며 검사하고자 하는 그룹 세트 명칭을 기입함.
  * **Name**: 확인 대상 속성/어트리뷰트의 이름(Name)을 기입함.
  * **Condition (Exists / Contains)**:
    * **Exists**: 대상 속성/물량 항목이 객체 내에 존재(누락되지 않음)하는지만 판정함. (선택 시 Value 입력창 비활성화).
    * **Contains**: 속성 항목이 존재해야 할 뿐만 아니라, 특정 값 조건을 만족해야 함을 지시함.
  * **Value**: Contains 선택 시 활성화되며 정합성을 만족시킬 타겟 문자 패턴 또는 텍스트를 기입함.
* **빌더 도구 모음 제어 버튼**:
  * **`Check` 버튼 (실행 아이콘)**: 현재 설정된 빌더 규칙대로 활성 모델 내 검증을 즉각 구동함.
  * **`Save` 버튼 (디스크 아이콘)**: 설계한 검증 규칙을 새 스펙으로 최종 패키징하여 상단의 **Spec. List** 목록에 정식 등록함.
  * **`Select` 버튼 (선택 아이콘)**: 직전 실행한 검증 결과에 포함된 전체 객체(Pass/Fail 전체)를 3D 뷰포트 상에서 한 번에 하이라이트 선택 처리함.
  * **`To Topic` 버튼 (BCF 연동 등록)**: 검증 결과 중 규칙을 위배한 **"Fail" 불합격 객체들만 전부 추출**하여 3D 카메라 뷰포인트 좌표, 반투명 상태, 위배 객체 빨간색(C00000) 강조 표현을 담은 BCF Topic 이슈로 실시간 자동 일괄 생성하고 목록에 신속히 등록함.
  * **`Export` 버튼 (CSV 추출)**: 쿼리 검사 결과로 얻은 전체 객체 판정 내역을 CSV 파일(`ids_check_results.csv`)로 내보냄.

### 8.5. Spec. Check Results (검사 판정 결과 리스트)
* 실행된 IDS 스펙 검사 조건에 의해 필터링된 모든 객체의 합격/불합격 현황을 테이블로 한눈에 검토하는 모니터링 영역임.
* **결과 컬럼 구성**:
  * **Name**: 검사 대상 부재의 IFC 고유 명칭.
  * **GUID**: 대상 부재의 고유 글로벌 식별자.
  * **Entity**: 대상 부재의 IFC 클래스 유형(예: IfcWall, IfcSlab 등)을 표시함.
  * **Value**: 검사 조건 대상 속성에 기입되어 있었던 실제 실시간 값 정보 (값이 아예 없거나 누락 시 `Null` 출력).
  * **Status (Pass / Fail)**: 검사 조건 충족 여부에 따른 합격 판정 뱃지.
    * **Pass (초록색)**: 속성 정보가 정상적으로 존재하고 값 일치 조건을 만족함.
    * **Fail (빨간색)**: 속성 자체가 누락되었거나 값 조건이 불일치하여 납품 규칙을 위배함.
* **행 클릭 연동 및 다크 고스트 피드백**
  * 결과 리스트의 특정 행을 마우스로 클릭하면, 3D 뷰포트 내에서 **해당 객체의 위치로 카메라가 즉시 자동 피팅(Fit to Items) 및 줌인 조준**되며, 선택된 객체 외의 다른 모든 모델들은 반투명(Ghost) 상태로 변화하여 오류 부재의 형상과 도면상의 위치를 입체적으로 식별하도록 시각을 집중시킴.

## 9. Properties 레이아웃 설명
* 본 플랫폼은 로드된 IFC 모델의 메타데이터 속성 정보를 단순히 조회하는 것을 넘어, 사용자가 브라우저 상에서 원하는 부재를 타겟팅하여 직접 프로퍼티 세트(Pset) 및 프로퍼티 항목(Property)을 실시간으로 추가, 수정 및 삭제할 수 있는 객체 속성 저작 도구를 제공함.

### 9.1. 섹션 구성
* **좌우 2단 분할 레이아웃**:
  * 속성 편집 도구와 원본 속성 테이블을 좌측에 배치하고, 우측의 3D 뷰포트와 실시간 연동하는 심플한 구조임.
  * **좌측 상단**: Properties Manager (프로퍼티 세트 및 개별 프로퍼티 키/값 추가 및 삭제 저작 양식 빌더)
  * **좌측 하단**: Selection Data (현재 선택된 객체의 원본 속성 확인 테이블)
  * **우측 전체**: 3D 뷰포트 (속성 편집 대상 객체 선택 및 실시간 파일 리로드 시각화)
* **유연한 크기 제어**:
  * 좌측의 두 패널(`propsManager`, `elementData`) 간의 높이 비율과 좌측 정보창 및 우측 3D 뷰포트 간의 가로 너비(Half column width) 비율을 드래그하여 직관적으로 세부 조정할 수 있음.

### 9.2. Properties Manager 섹션 - 프로퍼티 추가, 삭제, 수정을 위한 사용자 지침
* **Properties Manager 구성**:
  * 마우스 선택(Highlight) 또는 Express ID 입력을 통해 대상을 지정하고, 직접 작성한 프로퍼티 정보들을 일괄 매핑하는 빌더 섹션임.
* **프로퍼티 데이터 빌더 생성 과정**
  * **`+ Add Pset` 버튼**: 신규 프로퍼티 세트(Pset) 블록을 빌더 양식에 동적으로 추가함.
    * 블록 상단의 이름 입력란에 추가하고 싶은 Pset 명칭(예: `Pset_WallCommon`, `Custom_Pset` 등)을 직접 기재함.
    * 세트 블록 우측의 휴지통 아이콘 클릭 시, 작성 중이던 해당 Pset 명칭과 하부 속성들을 목록에서 일괄 제거함.
  * **`+ Add Property` 버튼**: 해당 Pset 블록 하위에 개별 프로퍼티 항목 행을 추가함.
    * 추가된 행의 좌측란에는 속성 명칭(Property Name), 우측란에는 속성에 대입할 값(Value)을 텍스트로 기입함.
    * 행 끝의 `X` 아이콘 클릭 시 해당 프로퍼티 입력 필드만 삭제 처리함.
* **프로퍼티 수정 및 저장 조작 지침**
  * **Apply Properties**:
    1. 3D 뷰포트에서 속성을 편집할 대상 부재를 마우스 클릭하여 선택하거나, 하부의 `Invisible Category Finder`(보이지 않는 공간용기 요소 분류 검색기)를 열어 대상 객체를 선택하여 Express ID를 지정함 (혹은 ID를 직접 쉼표 구분하여 입력란에 수동 기입 가능).
    2. 입력한 Pset 및 속성 빌더 조건 데이터들을 타겟 부재들에 주입(Add/Modify)하여 IFC 모델을 업데이트함.
    3. 내부 가공 연산을 마친 후, 속성이 주입 및 갱신된 새로운 IFC 모델 기반으로 3D 공간 상에 리로드(Reload)하여 화면을 갱신하고 선택 상태를 복구함.
  * **Delete Properties**:
    * 선택한 부재에서 빌더에 지정된 Pset이나 Property 조건에 해당하는 메타데이터 항목들만 선별 제거하고 모델을 리로드함.
  * **Download**:
    * 프로퍼티 수정이 임시 보관된 IFC 파일과 형상 최적화 파일(`.frag`)을 로컬 PC에 물리적으로 저장함. (기본적으로 `모델명_prop` 형태의 파일명이 제시되며 수정 가능).
  * **Save to DB**:
    * 프로퍼티 수정이 누적 완료된 최종 IFC 및 FRAG 파일을 Oracle DB 저장소에 영구 동기화하여 저장 및 덮어쓰기함.

### 9.3. 프로퍼티 관리 마이크로 서비스
* **마이크로서비스 구조**:
  * Node.js 백엔드 Express 서버(`src/app.ts`)의 `/api/process-properties` 엔드포인트와 Python 기반의 전용 경량 마이크로서비스(포트 `8000`, `/process-properties`) 간의 API 연동을 통해 구동됨.
* **동작 프로세스**:
  1. 프론트엔드에서 `Apply/Delete` 액션 호출 시, Node.js 서버는 임시 스토리지의 IFC 바이너리 데이터와 가공할 Express ID 및 JSON 속성 데이터를 묶어 Python 마이크로서비스로 전송함.
  2. Python 마이크로서비스는 `ifcopenshell` 라이브러리를 이용하여 업로드된 IFC 스키마를 정합적으로 파싱한 후, 타겟 객체(Express ID)에 대한 `IfcPropertySet` 생성 및 속성 정의 관계인 `IfcRelDefinesByProperties` 매핑을 파이썬 내부 엔진 단에서 엄격하게 추가/삭제 연산함.
  3. 데이터 수정이 완료된 새로운 완결적 IFC 파일 바이너리를 Node.js 서버에 최종 반환하여, 뷰어에 최신 속성이 내재된 IFC 파일이 즉각 리로드 및 저장되도록 데이터 연계 무결성을 보장함.

## 10. Quantities 레이아웃 설명
* 본 플랫폼은 3D 뷰포트에서 마우스 클릭 및 다중 선택한 부재들의 기하학적 치수 속성 데이터(길이, 면적, 체적, 개수 등)를 실시간 추출하고 분석하여 요약 및 차트 형태로 시각화하는 물량 정보 전용 레이아웃을 제공함.

### 10.1. 섹션 구성
* **아코디언 구조의 유연한 패널**:
  * Quantities 패널 섹션은 크게 Data Filters, Quantity Summary, Quantity Data, Quantity Chart의 4개 기능 구획으로 구성되어 있음.
  * 각 하부 구획 타이틀 영역을 마우스 클릭하여 개별적으로 축소/확장(Collapse/Toggle)할 수 있어 사용자가 보고자 하는 통계 데이터 영역을 유연하게 조절해 탐색이 가능함.
* **실시간 데이터 로딩**: 3D 뷰포트 상에서 객체 선택 변경(Highlight 및 Clear) 이벤트를 동적으로 추적하여, 해당 부재의 물량 속성을 즉각 백업 계산하고 타이틀 개수 정보(`Quantity Table (N)`)를 실시간 업데이트함.

### 10.2. Quantity Table - Data filter (데이터 필터)
* 검출 및 수집된 대량의 부재 물량 목록 중, 원하는 특정 범주의 텍스트 정보 또는 수치 구간 조건에 해당하는 부재들만 정밀 선별하는 필터 도구임.
* **Categorical Filters (범주형 필터)**:
  * 드롭다운 메뉴에서 문자 속성 키(Category, Name, PredefinedType 등)를 선택하고 텍스트 입력창(`Filter value...`)에 값을 입력하여 실시간 포함 조건(Contains) 검색을 처리함.
* **Numeric Filters (수치형 필터)**:
  * 드롭다운 메뉴에서 수량 유형(Volume, Area, Length 등)을 선택하고, 수치 입력란에 **최소값(Min)** 및 **최대값(Max)** 구간을 기입하여 해당 수치 범위 내에 드는 부재만 필터링함.
* **Clear All Filters 버튼 (`X` 아이콘)**: 현재 입력된 텍스트 조건 및 수치 범위 필터 값 전체를 일괄 초기화하여 목록을 원본 상태로 복구함.

### 10.3. Quantity Table - Quantity Summary (물량 요약)
* 현재 필터링을 거쳐 리스트에 남은 대상 부재들의 선택된 수량 속성값들을 수학적으로 종합하여 계산 요약 테이블 형태로 나타냄.
* **수량 요약 제어**:
  * 타이틀 옆의 속성 다중 선택 드롭다운(`Dropdown Multiple`)을 통해 집계 대상이 될 수량 항목(NetVolume, NetArea, Length 등)을 지정함.
* **집계 요약 정보**:
  * 선택된 수량에 대하여 활성 부재 전체의 **총합(`Sum`)**, **평균(`Mean`)**, **최대치(`Max`)**, **최소치(`Min`)** 통계값을 소수점 둘째 자리까지 반올림 연산하여 목록 형태로 깔끔하게 표시함.

### 10.4. Quantity Table - Quantity Data (상세 물량 데이터)
* 필터 조건 및 선택을 충족하는 개별 부재들의 모든 속성과 물량 값을 열(Column)과 행(Row)의 표 형식으로 상세 출력하는 스프레드시트 영역임.
* **동작 원리 및 데이터 파싱**:
  * IFC 모델의 `IsDefinedBy` 및 `IsTypedBy` 관계 스키마를 동적으로 추적하여 개별 객체의 LengthValue, AreaValue, VolumeValue, CountValue, NominalValue 등 수량 Pset 데이터를 정밀 추출함.
  * 칼럼 간 데이터 정합성을 깨뜨리지 않도록 값이 비어있거나 수량이 존재하지 않는 경우 대시(`-`)를 기입하여 채움.
* **CSV 내보내기 (`Export to CSV`)**:
  * 타이틀 우측의 내보내기 버튼(내보내기 아이콘) 클릭 시 현재 필터링되어 화면에 구성된 모든 물량 데이터 리스트를 로컬 PC에 CSV 파일(`Quantity_Data.csv`)로 간편하게 저장 및 다운로드함.
* **행 클릭 카메라 포커싱 연동**:
  * 상세 표 목록의 임의의 부재 행을 마우스로 클릭하면, 3D 뷰포트 내에서 **해당 부재의 위치로 카메라가 실시간 피팅(Fit to Items) 및 정렬**되어, 물량 수치와 3D 물리적 외형 구조를 동시에 실시간 대조 검토할 수 있도록 동작함.
* **Pagination (페이징)**:
  * 대용량 물량 계산 시 렌더링 지연을 미연에 방지하고자 페이지당 30개씩 분할하여 데이터를 로드함.

### 10.5. Quantity Table - Quantity Chart (물량 분포 차트)
* 필터가 적용된 부재들의 물량 분포 밀도를 시각적인 히스토그램 막대그래프로 구성하여 보여주는 통계 차트임.
* **차트 속성 전환**:
  * 차트 내에 포함된 전용 수량 타입 전환 컨트롤러를 조작하여 체적(Volume), 면적(Area), 길이(Length) 등 시각화하고 싶은 물량 속성 기준을 간편하게 변동함.
* **차트 조각 클릭 필터링 연동**:
  * 히스토그램의 특정 막대 조각을 마우스로 클릭하면, 해당 막대가 대변하는 **수량 범위의 최솟값과 최댓값 수치를 상부 데이터 필터(Numeric Filters)의 Min 및 Max 조건 범위 입력창으로 자동 주입 및 동기화** 처리함.
  * 주입 즉시 연동 필터링 계산을 거쳐, 해당 수량 범위에 포함되는 부재들만 `Quantity Data` 목록과 `Summary` 테이블에 나타나도록 화면을 지능적으로 업데이트함.

## 11. Timeline 레이아웃
* 본 플랫폼은 3D BIM 모델의 객체 정보와 시간 축(공정 데이터)을 연계하여 가상의 시공 단계별 4D 시뮬레이션을 구현하고 제어하는 Timeline 레이아웃을 제공함.

### 11.1. 섹션 구성
* **공정 관리 및 시뮬레이션 특화 레이아웃**:
  * Timeline 레이아웃은 공정 규칙을 편집하는 상부 구역과, 4D 애니메이션 및 간트 차트를 모니터링하는 하부 전체 구역으로 구성됨.
  * **상단 좌측**: 3D 뷰포트 (설정된 공정에 맞춰 부재의 설치 및 구축 상태를 3D 화면에 실시간 렌더링)
  * **상단 우측**: Phase Manager (시공 단계를 생성하고 특정 모델, 층, 카테고리별로 공정 시작일 및 공사 기간을 정의하는 규칙 편집 테이블)
  * **하단 전체**: Timeline (플레이어 컨트롤러 및 간트 차트 형식의 공정 시각화 패널)
* **드래그 리사이징**:
  * 마우스 드래그를 통해 상하단 높이 비율 및 상단 좌우(뷰포트와 Phase Manager) 너비를 유연하게 재조정하여 넓은 타임라인 영역을 쾌적하게 모니터링할 수 있음.

### 11.2. Phase Manager (공정 규칙 편집기)
* 모델의 속성 데이터를 추출하여 사용자가 정의한 규칙에 따라 공정 단계(Phase)를 생성하고 3D 부재들과 연계하는 핵심 규칙 편집기 패널임.
* **상단 제어 버튼 그룹**
  * **Generate Draft 버튼 (문서 아이콘)**: 로드된 활성 모델 데이터를 분석하여 `Model ➔ Storey(층) ➔ Entity(카테고리)` 계층 순서로 30일 간격의 시공 흐름을 갖는 전체 공정 초안(Draft)을 자동으로 일괄 빌드함. (기존 규칙 존재 시 덮어쓰기 경고창 노출).
  * **Add a Phase Rule 버튼 (`+` 아이콘)**: 수동으로 새로운 공정 규칙(Phase Rule) 행을 목록 하단에 추가함. (기본 7일 기간 및 오늘 날짜 기준 시작일 주입).
  * **Sort by Start Date 버튼 (정렬 아이콘)**: 등록된 모든 공정 규칙들을 시작일(Start Date) 기준으로 오름차순 정렬하고, 순서에 맞게 공정 시퀀스 번호(Phase)를 자동으로 재조정함.
  * **Apply Phase Rules 버튼 (Gantt 차트 아이콘)**: 편집한 공정 규칙들을 3D 모델 객체들과 매핑 연산하여 하위의 **Timeline** 간트 차트에 데이터를 최종 적용 및 반영함.
* **공정 규칙 편집 테이블 항목**
  * **Model**: 규칙을 적용할 특정 IFC 모델을 선택함. (All을 선택하면 모든 모델 공통 적용).
  * **Storey**: 선택한 모델에 속한 특정 층(Storey)을 필터링하여 매핑함. (All 지원).
  * **Entity**: 모델과 층 조건에 부합하는 특정 IFC 클래스 유형(Entity) 목록을 필터링하여 선택함. (All 지원).
  * **Object**: 상세 텍스트 입력을 통해 특정 이름을 가진 부재를 골라내 공정을 연결함. (기본값: All).
  * **Start**: 해당 공정 단계 of 공사 시작일(yyyy-mm-dd)을 날짜 피커를 통해 지정함.
  * **Days**: 해당 공정에 소요될 공사 기간(일수)을 수치로 지정함.
  * **Action (Remove Rule - 휴지통 아이콘)**: 해당 행의 공정 규칙을 리스트에서 즉시 제거함.
* **모델 이벤트 연동**:
  * 새로운 모델이 로드되거나 기존 모델이 메모리에서 해제(Dispose)될 경우, Phase Manager 내부의 모델/층/카테고리 드롭다운 옵션 및 기존 공정 룰이 실시간 자동 갱신 및 리셋 처리됨.

### 11.3. Timeline (공정 시뮬레이터 및 Gantt 차트)
* Phase Manager에서 적용한 공정 정보를 기반으로, 시간 흐름에 따른 3D 부재 조립 및 시공 과정을 4D 애니메이션으로 제어하고 모니터링하는 플레이어 패널임.
* **플레이어 제어 도구 모음**
  * **Play 버튼 (재생 아이콘)**: 시공 시뮬레이션을 시작함. 전체 프로젝트 공사 기간을 기준 비율로 10초 동안 압축하여 3D 뷰포트에 부재가 점진적으로 완성되는 애니메이션을 시각적으로 시연함.
  * **Pause 버튼 (일시정지 아이콘)**: 현재 재생 중인 공정 시뮬레이션 애니메이션을 즉시 일시 정지시킴.
  * **Repeat 버튼 (반복 아이콘)**: 무한 반복(Loop) 재생 옵션을 설정함. 활성화 시 공정이 끝까지 도달하면 처음 시공 시점으로 되돌아가 자동으로 재생을 반복함.
* **동적 시각 피드백 및 스크러빙**
  * 하단에 렌더링된 간트 차트 영역의 슬라이더 바를 사용자가 직접 드래그(Scrubbing)하여 원하는 특정 공사 시점(Progress)으로 즉시 건너뛰거나 진행 상황을 실시간 제어할 수 있음.
  * 진행률 변동 시, 해당 시점 이전까지의 시공이 완료된 3D 모델 부재들만 화면에 가시화(`showElements`) 처리하여 시간별 공사 현황을 입체적으로 관측 가능함.
  * 모델이 전부 비워지면(Dispose) 애니메이션 재생을 강제 중지하고 진행 상태를 초기 상태(0)로 안전하게 자동 리셋함.

## 12. FullScreen 레이아웃
* 본 플랫폼은 복잡한 데이터 조회 패널이나 입력 폼들을 모두 숨기고, 오직 3D 모델의 시각화 및 집중적인 그래픽 탐색만을 위해 화면 전체를 3D 뷰포트로 채우는 FullScreen 레이아웃을 지원함.

### 12.1. 섹션 구성
* **단일 뷰포트 중심의 전체 화면 구성**:
  * FullScreen 레이아웃은 여타 사이드 패널(IFC List, Model Tree, Selection Data 등)을 포함하지 않는 레이아웃 구조임.
  * **전체 영역**: 3D 뷰포트 (Viewer) 하나만 화면 비율에 맞춰 100% 꽉 차게 렌더링됨.
  * 패널 간의 경계(Gap)가 없어 리사이징 드래그 바 역시 활성화되지 않음.

### 12.2. 전체화면 3D 뷰포트 활용
* **집중 검토 및 프레젠테이션 환경**:
  * 복잡한 속성 테이블이나 공정 차트 등의 UI 요소를 화면에서 완전히 제거하여, 의사결정 회의나 프로젝트 시연(프레젠테이션) 시 대형 화면에 모델만 투사하여 검토하는 데 최적화되어 있음.
  * **3D 뷰포트 조작**: 마우스 왼쪽 버튼 드래그로 회전(Orbit), 오른쪽 버튼 드래그로 이동(Pan), 휠 스크롤로 줌인/아웃(Zoom)을 수행하는 표준 뷰어 조작계를 동일하게 지원함.
  * **뷰포트 선택 상태 유지**: 이전 레이아웃에서 객체를 선택하거나 하이라이트한 상태(Selection)가 전체화면 모드로 변경되더라도 그대로 유지되어, 특정 객체를 집중 조명한 상태에서 대화형 모니터링이 가능함.
